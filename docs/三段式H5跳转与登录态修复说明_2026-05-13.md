# 三段式 H5 跳转与登录态修复说明（2026-05-13）

## 背景

三段式链路：

```text
入口域名 ls.hnstylor.cn
  -> 授权域名 ls.tyliao.cn
  -> 主站域名 ls.hainanjunfeng.com
```

线上反馈两个问题：

1. 三段式跳转后，主站 `ls.hainanjunfeng.com` 没有稳定拿到 `merchantId/userId`，导致推广者缺失。
2. 用户登录后再次扫码，仍然进入登录页，表现像 token 没保留。

团队确认：商家主页类入口不再跳 `/enter`，应直接进入：

```text
/webh5/jhhome
```

## 修复内容

### 1. 三段式目标页调整为直达 `/jhhome`

涉及文件：

```text
src/utils/threeStepRedirect.js
tests/threeStepRedirect.test.mjs
```

当前规则：

```text
无 contentId：/webh5/jhhome?merchantId=xxx&userId=xxx
有 contentId：/webh5/contentDetail?contentId=xxx&userId=xxx
```

避免再次走 `/enter` 带来的额外跳转和参数不稳定。

### 2. 跨域回跳 redirect 参数保留

涉及文件：

```text
src/utils/threeStepRedirect.js
src/views/auth-callback/index.vue
```

新增/调整能力：

- `sanitizeExternalRedirectPath(...)`
  - 将外部主站 URL 转为站内 path，例如：

```text
http://ls.hainanjunfeng.com/webh5/jhhome?merchantId=100&userId=200
-> /jhhome?merchantId=100&userId=200
```

- `normalizeInternalRedirectPath(...)`
  - 防止 `//evil.example`、`http://evil.example`、`/login` 等异常 redirect。

- `normalizeInternalRedirectLocation(...)`
  - 给 Vue Router 使用，避免把 `/jhhome?merchantId=1` 错误当作 `path` 导致 query 丢失。

### 3. 修复已登录再次扫码仍进登录页

涉及文件：

```text
src/views/login/index.vue
src/stores/user.js
src/utils/request.js
src/main.js
src/router/index.js
src/utils/authStorage.js
```

核心原因：

```text
localStorage 是按域名隔离的。
ls.tyliao.cn 的 token 不能直接被 ls.hainanjunfeng.com 读取。
跨域登录必须通过 ticket 传递登录态。
```

修复后流程：

```text
用户再次扫码
  -> ls.hnstylor.cn /qiaoliao
  -> ls.tyliao.cn /login
  -> 授权域发现本域已有 token
  -> 调 /api/user/auth/ticket/create 创建 ticket
  -> 跳 ls.hainanjunfeng.com /auth-callback?ticket=xxx&redirect=/jhhome?merchantId=xxx&userId=xxx
  -> 主站兑换 ticket
  -> 主站写入本域 localStorage token
  -> 进入 /jhhome 并保留 query
```

### 4. 登录态本地恢复集中化

新增文件：

```text
src/utils/authStorage.js
```

作用：

- 统一读写 `token/userInfo`。
- Pinia 初始化时恢复 token。
- 路由守卫和请求拦截器每次使用前同步一次本地 token。

避免页面刷新或跨域回跳后，`localStorage` 有 token 但 Pinia 内存态未恢复。

## Review 重点

1. 无 `contentId` 的入口必须直达 `/jhhome`。
2. 有 `contentId` 的文章入口必须进入 `/contentDetail`。
3. `merchantId/userId` 必须从入口域传到授权域，再通过 redirect/ticket 保留到主站。
4. 已登录再次扫码不应停留在登录页。
5. `normalizeInternalRedirectLocation` 必须保留 query，避免推广者丢失。
