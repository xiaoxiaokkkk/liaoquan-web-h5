# 首页路由改造检查清单

> 适用场景：将首页路由从旧配置（如 `/index` + `Index`）改为新配置（如 `/first` + `First`）。

## 1. 先确定目标

- 新首页 `path`：例如 `'/first'`
- 新首页 `name`：例如 `'First'`
- 是否保留旧首页兼容路由：如 `'/home'`（建议评估后决定）

---

## 2. 必改文件

## `src/router/router.config.js`

- 根路由默认重定向
  - `redirect: '/xxx'` 改为新首页 path
- 首页路由定义
  - `path` 改为新首页 path
  - `name` 改为新首页 name
- 若有别名或兼容路由（如 `/home`），确认是否继续保留

## `src/router/index.js`

- 登录后默认跳转兜底地址
  - 例如 `redirectPath` 的 fallback 改为新首页 path
- 路由守卫中首页特殊判断
  - 例如 `to.name === 'Index'` 改为新首页 name
  - 若兼容旧路由名，写成并列判断（如 `Home || First`）

---

## 3. 高概率需要同步的文件

## `src/views/layouts/index.vue`

- `TabBar` 首页项是否按 `name` 跳转（如 `{ name: 'First' }`）
- 首页项的 `name` 字段是否与新首页 name 一致

## `src/components/TabBar.vue`

- 默认激活值
  - 如 `const activeName = ref('First')`
- 首页分支判断
  - 如“切回首页补 query”逻辑中，是否还在判断旧 name

## 业务页面（例如 `src/views/contentdetail/index.vue`）

- 返回首页的硬编码地址
  - `router.push('/index?...')` / `router.replace('/index?...')`
  - 统一改成新首页 path

---

## 4. 全局检索关键词（防漏改）

建议至少全局搜以下关键词：

- `'/index'`
- `'/index?'`
- `'Index'`
- `name: 'Index'`
- `to.name === 'Index'`
- `router.push(`、`router.replace(`（检查是否写死旧首页 path）

如果你是改成其他名字，把上面旧值替换成你当前项目的旧首页配置即可。

---

## 5. 建议改造顺序

1. 先改路由配置（`router.config.js`）
2. 再改守卫和默认回跳（`router/index.js`）
3. 再改 TabBar 与布局入口（`layouts`、`TabBar`）
4. 最后全局清理业务页面硬编码跳转
5. 做一次全局检索复核

---

## 6. 回归测试清单（最少执行）

- 直接访问根路径 `/`，是否正确落到新首页 path
- 登录后自动跳转，是否落到新首页
- TabBar 点“首页”，是否高亮正确、是否带回必要 query 参数
- 从内容详情页返回首页，是否跳到新首页且参数正确
- 浏览器前进/后退是否正常（不出现循环跳转）
- 刷新新首页 URL 后，页面是否可正常恢复

---

## 7. 常见问题

- 只改了 `path` 没改 `name` 引用：TabBar/守卫逻辑可能失效
- 只改了 `name` 没改硬编码 `path`：业务页面跳转仍走旧地址
- 兼容路由（如 `/home`）存在时，逻辑判断只写了一个 name，导致部分场景不生效

---
